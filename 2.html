CSS

CSS는 레이아웃 , 색상 및 글꼴을 포함하여 프레젠테이션과 콘텐츠를 분리 할 수 ​​있도록 설계되었습니다 . [3] 이러한 분리는 콘텐츠를 향상시킬 수있는 접근성 , 프리젠 테이션 특성의 사양에 더 많은 유연성과 제어를 제공, 여러 수 있도록 웹 페이지를 별도의 .css 파일에 관련 CSS를 지정하여 서식을 공유하고, 구조적 내용의 복잡성과 반복을 감소 . 서식과 내용을 분리하면 화면 상, 인쇄 중, 음성 (음성 기반 브라우저 또는 스크린 리더를 통해 ), 점자 기반 (Braille-based ) 과 같은 다른 렌더링 방법에 대해 서로 다른 스타일로 동일한 마크 업 페이지를 제공 할 수도 있습니다 촉각 장치. CSS에는 모바일 장치 에서 콘텐츠에 액세스하는 경우 대체 서식 지정 규칙이 있습니다 . [4] 이름 계단식 은 지정된 우선 순위 체계에서 비롯되어 두 개 이상의 규칙이 특정 요소와 일치하는 경우 적용 할 스타일 규칙을 결정합니다. 이 계단식 우선 순위 체계는 예측 가능합니다. CSS 사양은 W3C ( World Wide Web Consortium )에서 관리합니다. 인터넷 미디어 유형 ( MIME 유형 ) text/css은 RFC 2318 (1998 년 3 월)에 따라 CSS와 함께 사용하도록 등록됩니다 . W3C는 CSS 문서에 대해 무료 CSS 유효성 검사 서비스 를 운영합니다 . [5] HTML 외에도 다른 마크 업 언어는 XHTML , 일반 XML , SVG 및 XUL을 비롯한 CSS 사용을 지원합니다 .
